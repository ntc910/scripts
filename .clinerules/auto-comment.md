## Brief overview
Guidelines for automatically inserting a comment marker into any code or text that is generated by Cline (the AI assistant) within this project. The rule ensures generated content is clearly identified, making it easy to review, audit, and manage AI‑produced sections.

## Automatic comment insertion
- Every snippet that originates from Cline must be preceded (or followed) by a comment indicating its source.
- The comment is added **immediately** after the generation event, before any further user edits or saves.
- The comment must remain in the file until a developer explicitly removes it.

## Trigger conditions
- When a Cline‑related command writes text to a file
- When Cline inserts text through a chat‑based suggestion or any custom extension command that calls the Cline service.
- The rule does **not** apply to manually typed code or edits that are not produced by Cline.

## Comment format per language
| Language / File type | Comment syntax | Example |
|----------------------|----------------|---------|
| TypeScript / JavaScript | `// [Cline generated]` | `// [Cline generated]\nconst foo = 42;` |
| Python | `# [Cline generated]` | `# [Cline generated]\nfoo = 42` |
| HTML / XML | `<!-- [Cline generated] -->` | `<!-- [Cline generated] -->\n<div></div>` |
| CSS / SCSS | `/* [Cline generated] */` | `/* [Cline generated] */\nbody { margin:0; }` |
| Markdown | `<!-- [Cline generated] -->` | `<!-- [Cline generated] -->\n# Title` |
| JSON | `/* [Cline generated] */` (placed before the opening `{` on its own line) | `/* [Cline generated] */\n{ "key": "value" }` |

## Integration helper
Create a utility function `addClineComment(uri: vscode.Uri, generatedText: string, languageId: string): Promise<void>` that:
1. Determines the correct comment syntax from `languageId`.
2. Prepends the comment line to `generatedText`.
3. Writes the combined text to `uri` using the appropriate VS Code file‑system API.

All Cline‑related commands in this repository (e.g., those in `EditTracker`, `GitIntegrationService`, or any future AI‑generation commands) should use this helper when persisting generated content.

## Review workflow
- The VS Code status bar or dashboard should highlight files containing the `[Cline generated]` marker, allowing quick navigation.
- During code reviews, reviewers can filter for the marker string to focus on AI‑generated sections.
- A VS Code command `clineStats.showGeneratedComments` can open a quick‑pick list of all files/sections that contain the marker.

## Documentation & maintenance
- Add a reference to this rule in the project README with a short note explaining its purpose.
- If a developer wishes to disable automatic commenting for a specific file, they may add the file path to the setting `clineStats.disableAutoComment` (array of glob patterns) – the helper function must respect this setting.
- Periodically run a lint step that warns if any generated block lacks the comment marker.

## Edge cases
- If a generated block is immediately followed by another generated block, each block receives its own comment line.
- When a generated block is later edited manually, the comment remains unless the developer removes it.
- For binary or non‑text files, the rule does not apply.

## Auto‑comment removal on Git commit
When a Git commit command is executed, the extension automatically scans the workspace and removes any `[Cline generated]` comment markers. This ensures the repository does not contain generated markers, keeping the codebase clean.

## Cline Generated Code Marking

### Mandatory Flag
- All code generated by Cline MUST be marked with the `[Cline generated]` flag
- The flag should be placed as a comment at the top of the file or before the specific code block generated by Cline

### Examples

#### At the file level:
```typescript
// [Cline generated]
import React from 'react'

export function MyComponent() {
  // Component code...
}
```

#### Before specific code blocks:
```typescript
// [Cline generated]
const fetchData = async () => {
  // Generated function implementation
}
